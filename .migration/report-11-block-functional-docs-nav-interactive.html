6. Sidebar Navigation

3 VariationsComplexity: MediumEst: 3–5 dev daysPriority: P1
Block Overview: Left sidebar navigation for category/sub-category page hierarchies. Provides contextual wayfinding within a section. Found on all Legacy category pages and some Modern pages. Features collapsible sections with expand/collapse icons.

V1: Modern Collapsible Sidebar

Modern

Left sidebar with expandable category sections using AEM Core accordion pattern. Smooth CSS transitions on expand/collapse. Current page highlighted.

Structure / Fields

ElementTag/ClassDescription
Container.cmp-p-sidebar-navSidebar wrapper, typically in 25% column
Section Header.cmp-p-sidebar-nav__headerClickable section header with toggle
Link Listul > li > aNested navigation links
Active State.activeCurrent page indicator

Acceptance Criteria

  • Sidebar renders in left column (25% width)
  • Sections expand/collapse on click with smooth animation
  • Current page is visually highlighted (bold/colored)
  • Nested levels properly indented
  • On mobile, sidebar collapses into an expandable drawer
  • Multiple sections can be open simultaneously

Interactions

Click (Header)
Toggles section open/closed. Chevron icon rotates 90→180 degrees.
Click (Link)
Navigates to the linked page. Current link gets active styling.
Keyboard
Enter/Space toggles sections. Arrow keys navigate between links.

Accessibility

  • Navigation region uses <nav> with aria-label="Section navigation"
  • Expand/collapse buttons have aria-expanded state
  • aria-controls links button to controlled section
  • Keyboard navigable with arrow keys

Reference URLs

bioprocessing.html

7. Accordion

2 VariationsComplexity: SimpleEst: 1–2 dev daysPriority: P1
Block Overview: Expandable/collapsible content sections for FAQs, support topics, and content organization. Two implementations: Modern AEM Core Components and Legacy Bootstrap-based.

V1: Modern AEM Core Accordion

Modern

AEM Core Components accordion with smooth expand/collapse animations. Chevron icon rotates on state change. Full ARIA support built-in.

Structure / Fields

ElementTag/ClassDescription
Container.cmp-accordionAccordion wrapper
Item.cmp-accordion__itemSingle accordion panel
Header.cmp-accordion__header buttonClickable toggle button
Icon.cmp-accordion__iconChevron/arrow indicator
Panel.cmp-accordion__panelCollapsible content area

Acceptance Criteria

  • Clicking header toggles panel open/closed
  • Smooth height animation on expand/collapse (300ms ease)
  • Chevron rotates 180° when expanded
  • Multiple panels can be open simultaneously (configurable)
  • First panel optionally pre-expanded on load
  • Content inside panels renders correctly (rich text, images, lists)

Interactions

Click
Toggle panel open/closed. Chevron animates.
Keyboard
Enter/Space: toggle. Up/Down arrows: navigate between headers. Home/End: first/last header.

Accessibility

  • role="region" on each panel
  • aria-expanded="true/false" on header buttons
  • aria-controls linking header to panel
  • aria-labelledby linking panel to header
  • Full keyboard navigation per WAI-ARIA Accordion Pattern

Reference URLs

support.html

V2: Legacy Bootstrap Accordion

LegacyNEW

jQuery Bootstrap collapse plugin. Uses panel-group/panel-heading/panel-collapse class structure. Plus/minus text toggle instead of chevron. Found on older category pages.

Acceptance Criteria

  • Bootstrap collapse functionality works
  • Plus changes to minus on expand
  • Panel content reveals smoothly
  • Only one panel open at a time (accordion mode)

Reference URLs

pcr.html (FAQ section)

8. Tabs

3 VariationsComplexity: MediumEst: 2–4 dev daysPriority: P1
Block Overview: Tabbed content interfaces for organizing related content into switchable panels. Three distinct implementations ranging from Modern AEM Core to Legacy workflow steppers.

V1: Modern Horizontal Tabs

Modern

AEM Core tabs with horizontal tab bar, active tab indicator underline, and smooth content transition between panels.

Acceptance Criteria

  • Tab bar renders horizontally with clear active indicator
  • Clicking tab switches visible panel instantly
  • Only one panel visible at a time
  • Tab content supports rich content (text, images, nested blocks)
  • Active tab has visual underline/highlight

Accessibility

  • role="tablist" on tab container
  • role="tab" on each tab button
  • role="tabpanel" on each content panel
  • aria-selected="true" on active tab
  • Arrow keys navigate between tabs

V2: Legacy Vertical Tabs

Legacy

Vertical tab list on the left, content panel on the right. jQuery-driven tab switching. Used for product comparison or specification tables.

Acceptance Criteria

  • Vertical tab list renders on left (~20% width)
  • Content panel on right (~80% width)
  • Active tab visually connected to panel (no gap/border)
  • On mobile, converts to accordion or horizontal tabs

V3: Workflow Tab Container (Stepper)

LegacyNEWComplex

Tabbed workflow stepper showing sequential process steps (e.g., molecular cloning workflow). 5 steps with visual progression indicators. Each tab reveals step-specific content with products and resources. Very content-rich (78 h3 headings, 108 images on the reference page).

9. Video

4 VariationsComplexity: MediumEst: 3–5 dev daysPriority: P1
Block Overview: Video blocks embed and play video content using Brightcove video.js platform primarily, with some YouTube embeds. Variations include inline players, modal/lightbox players, and playlist interfaces.

V1: Brightcove Inline Player

Modern

Embedded Brightcove video.js player rendered inline within content. Features auto-poster image, play button overlay, and full controls bar. Responsive 16:9 aspect ratio.

Acceptance Criteria

  • Player renders with poster image before playback
  • Play button centered on poster
  • Controls bar shows: play/pause, progress, volume, fullscreen
  • Responsive 16:9 container (no black bars)
  • Video loads on demand (not auto-play)

Interactions

Click (Play)
Poster image replaced by video. Playback starts. Controls appear on hover.
Controls
Standard video controls: play/pause, seek, volume, captions toggle, fullscreen.

Accessibility

  • Closed captions/subtitles available via CC button
  • Keyboard: Space=play/pause, arrows=seek, M=mute, F=fullscreen
  • Video player has aria-label with video title
  • Transcript option available for screen readers

V2: YouTube Embed

Both Platforms

YouTube iframe embed with privacy-enhanced mode (youtube-nocookie.com). Standard YouTube controls and responsive wrapper.

Acceptance Criteria

  • YouTube iframe renders within responsive container
  • Privacy-enhanced mode URL used
  • Standard YouTube player controls available
  • Responsive sizing maintained

V3: Video with Transcript

Modern

Video player with expandable transcript section below. Accordion-style expand for full transcript text, improving accessibility.

Acceptance Criteria

  • Video player renders above transcript area
  • Transcript section collapsed by default
  • Expand/collapse toggle labeled "View Transcript"
  • Transcript text is selectable and searchable

V4: Playlist Player

ModernNEWComplex

Brightcove video.js player on the left with a scrollable playlist panel on the right. Playlist shows video thumbnails for navigation. Features modal dialog, poster image, slider bar, and progress bar.

V1: Filterable Event Listing

ModernComplex

Single-column HTML table with full-width event card rows. 18+ event rows, each containing title (h2>a link), date/location (h4), description paragraph, and thumbnail image. Features text search + 3 checkbox filter groups (Event Type: 9 options, Region: 3 options, Month: 5 options = 17 total checkboxes).

Acceptance Criteria

  • All 18+ events display as card rows in the table
  • Text search filters events in real-time (keyup debounced)
  • Filters toggle button shows/hides the filter panel
  • Checking filter checkboxes immediately filters visible events
  • Multiple filters combine with AND logic within group, OR between groups
  • Each event card shows: title, date, location, description, thumbnail, CTA links
  • "No results" message when filters yield zero events
  • Filter state can be cleared with a "Reset" action

Interactions

Checkbox Filter
Immediate filtering on check/uncheck. Visual count of active filters.
Event Card Click

Filter Toggle
Shows/hides the filter panel with slide animation.

< 768px

  • Thumbnail hidden or smaller
  • Search bar full-width
Tablet

Desktop
> 1024px
  • Filter panel inline below search
  • Full card layout with right-aligned thumbnails

Accessibility

  • Search input has associated label and aria-label
  • Filter checkboxes have visible labels

  • Live region (aria-live="polite") announces result count changes
  • Event links have descriptive text

Modern

Horizontal sticky bar that follows scroll. Contains anchor links to page sections. Active state updates on scroll via IntersectionObserver. Smooth scroll to section on click.

Acceptance Criteria

  • Smooth scroll to target section on click

Both

Simple horizontal bar with pipe-separated links to page sections. Not sticky. Fixed position at section top.

Acceptance Criteria

  • Not sticky (scrolls with content)

LegacyNEW

Pipe-separated inline links serving as mini-directory. Used in order/catalog sections for alphabetical or category navigation.

Acceptance Criteria

  • Links render inline with visual separators
  • Each link navigates to corresponding section/page
  • Wraps gracefully on narrow viewports

15. Table

2 VariationsComplexity: SimpleEst: 1–2 dev daysPriority: P2

V1: Data Table

Modern

Standard HTML table with thead/tbody structure. Optional column sorting. Zebra striping for readability. Responsive horizontal scroll on mobile.

Acceptance Criteria

  • Table renders with proper thead/tbody structure
  • Zebra striping applied to alternating rows
  • Horizontal scroll wrapper on mobile (no content clipping)
  • Column headers are sticky if table is tall
  • Optional: click column header to sort

Accessibility

  • scope="col" on th elements
  • Caption or aria-label describing table purpose
  • Sort buttons have aria-sort state

V2: Comparison Table

Both

Multi-column product/feature comparison table. Header row fixed on scroll. Checkmark/X cells for feature presence. Hover highlights entire column.

Acceptance Criteria

  • Column header row sticky on vertical scroll
  • Checkmark (✓) and X (✗) cells rendered consistently
  • Column hover highlight for easy comparison
  • Product names in header link to product pages
  • Mobile: horizontal scroll with first column fixed
